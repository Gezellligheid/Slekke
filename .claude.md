# Slekke

## Wiki Knowledge Base

This project has a persistent wiki at `../slekke-docs` (relative to this directory).

**Always start a session by reading the wiki hot cache:**

1. Read `../slekke-docs/wiki/hot.md` first (~300 tokens, recent context summary)
2. If you need more: read `../slekke-docs/wiki/index.md` (full catalog)
3. For a specific module: read `../slekke-docs/wiki/modules/_index.md`, then the relevant page
4. Only read individual pages when you need depth on that specific topic

Do NOT read the full codebase if the wiki already has the context. The wiki exists to avoid that.

**After significant work in a session:**

1. Update `../slekke-docs/wiki/hot.md` with current context (overwrite completely, ~300-500 words)
2. Create or update module pages in `../slekke-docs/wiki/modules/` for anything built or changed
3. File architectural decisions in `../slekke-docs/wiki/decisions/`
4. Append a log entry to `../slekke-docs/wiki/log.md` (new entries at the TOP)

**Ingest a file into the wiki:**
Drop it in `../slekke-docs/.raw/code-dumps/` and say "ingest [filename]" — Claude will read, summarize, cross-reference, and file it.
